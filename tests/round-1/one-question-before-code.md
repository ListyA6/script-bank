# One question before I write a single line of code

> Before starting any automation build, Listy asks the operator one question — "what breaks when you're not here?" — and the answer decides whether the project is even worth doing.

2. Most operators can't answer this. Which is exactly the problem.
3. I've walked out of two client meetings before touching my laptop.
4. The question I ask every operator tells me if the software will actually get used.
5. I spent six months building things nobody needed. Now I ask one question first.
6. If you can't answer this in thirty seconds, I'm not writing anything yet.

## Voiceover — clean read
<!-- The full spoken track, exactly the way it's said. Read it aloud once. -->

Before I write a single line of code, I ask one question.

I build custom software for restaurants, shops, studios. And most owners come to me with a list of features they want. I used to just build the list. Three months in, the system's live, and nobody's using it right because the owner is still the one keeping it together.

So now I ask first.

"What breaks when you're not here?" That's it. And honestly, the first answer is almost never the real one. They say "the orders slow down." I ask again. They say "the cook doesn't know the prep order." I ask again. And then they say something like — "everything, basically."

That's the thing I'm actually building a system for.

Because if the answer is "nothing really breaks," then the software won't either. They don't need code, they need a checklist. And if the answer is "everything breaks," now I know where the real bottleneck is, and I can actually fix it.

Two operators this month couldn't answer it. I gave them homework before I touched my laptop.

## Post caption + hashtags

Before I build anything, I ask one question.

"What breaks when you're not here?"

Most operators can't answer it in thirty seconds. That's not a problem with their business — it's the map. The bottleneck is wherever the answer gets complicated.

Built a system for a studio last month. First answer was "bookings." Real answer, after two more rounds: "my staff doesn't know how to handle a reschedule without calling me." That's what I built for.

If you run a physical business and you're still the answer to that question — that's the thing to fix first.
